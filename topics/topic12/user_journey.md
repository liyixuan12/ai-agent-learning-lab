# 用户旅程（User Journey）

## 1. 主路径（Happy Path）


| 阶段      | 用户行为                    | 系统行为             | 用户获得的价值      | 风险点        |
| ------- | ----------------------- | ---------------- | ------------ | ---------- |
| 1. 进入任务 | 输入“比较 A 公司与 B 公司的财务健康度” | 识别意图、判定路由        | 快速开始，无需复杂提示词 | 意图识别错误     |
| 2. 信息澄清 | 用户补充时间范围/维度             | 系统给澄清问题模板        | 问题更精确，减少来回沟通 | 澄清过多导致流失   |
| 3. 执行分析 | 用户确认后触发分析               | 调用工具/检索/聚合       | 自动完成重复劳动     | 数据源不足或调用失败 |
| 4. 输出结果 | 查看结构化对比结论               | 返回结论 + 引用 + 免责声明 | 结果可复用，可追溯    | 无引用导致信任下降  |
| 5. 后续行动 | 追问“下一步看什么”              | 给教育性建议与下一步检查项    | 帮助形成研究路径     | 越界到投资建议    |


---

## 2. 异常路径（Failure Paths）


| 场景       | 用户感知        | 系统处理策略        | 兜底文案                          |
| -------- | ----------- | ------------- | ----------------------------- |
| 提示词注入    | 回答不稳定或异常自信  | 输入护栏拦截 + 安全降级 | “我不能执行越权或不安全指令，但可继续提供公开信息分析。” |
| 工具调用超时   | 长时间无响应      | 超时重试 + 部分结果返回 | “部分数据暂不可用，已先给出可验证结论与缺失项。”     |
| 信息不足     | 结果泛泛        | 触发澄清问题        | “为保证分析质量，请补充行业/时间区间/比较维度。”    |
| 用户请求违规建议 | 用户期待“买/卖结论” | 政策拒答 + 教育性替代  | “我不能提供个性化投资建议，但可解释相关风险因素。”    |


---

## 3. 关键体验指标映射

- **首轮可用输出率**：阶段 1-4 能否在首轮给到“可用初稿”。
- **任务完成时长**：阶段 1 到阶段 5 的总时长。
- **可追溯率**：阶段 4 结论带引用的比例。
- **安全拒答准确率**：异常路径中拒答应拒尽拒、不过度误拒。

---

## 4. 旅程优化建议（迭代优先级）

1. 先优化阶段 2 的澄清模板（降低流失）。
2. 再优化阶段 4 的输出结构（提高复用性）。
3. 最后做阶段 5 的追问引导（提升留存和深度使用）。
